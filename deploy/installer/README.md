# Dockerized coJournalist Installer

This is an optional third setup path for newsrooms that want a disposable
operator container instead of installing Node, Deno, Supabase CLI, and GitHub
CLI on the host.

The container does not define a separate setup flow. It mounts the same
`cojournalist-setup.json` generated by `/setup` and runs
`automation/setup-from-manifest.sh` from the repository.

## Build

```bash
docker build -f deploy/installer/Dockerfile -t cojournalist-installer .
```

## Install

From an existing checkout or an empty deployment directory:

```bash
docker run --rm -it \
  -v "$PWD:/workspace" \
  -v "$PWD/cojournalist-setup.json:/config/cojournalist-setup.json:ro" \
  cojournalist-installer install
```

If `/workspace` is already a coJournalist checkout, the installer uses it. If
not, it clones `buriedsignals/cojournalist-os` into
`/workspace/cojournalist-os`.

## Doctor

```bash
docker run --rm -it \
  -v "$PWD:/workspace" \
  cojournalist-installer doctor
```

## Update

```bash
docker run --rm -it \
  -v "$PWD:/workspace" \
  -v "$HOME/.config/gh:/root/.config/gh:ro" \
  cojournalist-installer update
```

`update` fetches upstream `master`, prepares a maintenance branch, refreshes
the weekly sync workflow, and opens a PR when GitHub CLI auth is mounted.
