# Dockerized coJournalist Installer

This is the recommended self-host setup path for newsrooms that want a
disposable operator container instead of installing Node, Deno, Supabase CLI,
GitHub CLI, jq, and OpenSSL on the host.

The container does not define a separate setup flow. It mounts the same
`cojournalist-setup.json` generated by `/setup` and runs
`automation/setup-from-manifest.sh` from the repository.

The same container is also the downstream maintenance operator: use it for
`doctor` before and after changes, and for `update` when preparing upstream
maintenance PRs in newsroom forks.

## Install

Run from the directory that contains `cojournalist-setup.json`. If `/workspace`
is not already a coJournalist checkout, the image clones
`buriedsignals/cojournalist-os` into `/workspace/cojournalist-os`.

```bash
docker run --rm -it \
  -v "$PWD:/workspace" \
  -v "$PWD/cojournalist-setup.json:/config/cojournalist-setup.json:ro" \
  ghcr.io/buriedsignals/cojournalist-installer:latest install
```

For a local development build instead of the prebuilt image:

```bash
docker build -f deploy/installer/Dockerfile -t cojournalist-installer .
docker run --rm -it \
  -v "$PWD:/workspace" \
  -v "$PWD/cojournalist-setup.json:/config/cojournalist-setup.json:ro" \
  cojournalist-installer install
```

## Doctor

```bash
docker run --rm -it \
  -v "$PWD:/workspace" \
  -v "$PWD/cojournalist-setup.json:/config/cojournalist-setup.json:ro" \
  ghcr.io/buriedsignals/cojournalist-installer:latest doctor
```

## Update

Run from the downstream newsroom fork checkout. Mounting an empty deployment
directory is fine for initial install, but update PRs need a fork checkout with
a pushable `origin`.

```bash
docker run --rm -it \
  -v "$PWD:/workspace" \
  -v "$PWD/cojournalist-setup.json:/config/cojournalist-setup.json:ro" \
  -v "$HOME/.config/gh:/root/.config/gh:ro" \
  ghcr.io/buriedsignals/cojournalist-installer:latest update
```

`update` fetches upstream `master`, prepares a maintenance branch, refreshes
the weekly sync workflow, and opens a PR when GitHub CLI auth is mounted.
Secrets stay out of Git. The manifest remains local and mounted read-only.
