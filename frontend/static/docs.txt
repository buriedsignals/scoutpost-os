# Scoutpost docs in text form

This is the short plain-text gateway to the public documentation. The full rendered docs live at `https://www.scoutpost.ai/docs`, and the flattened one-shot ingestion file lives at `https://www.scoutpost.ai/llms-full.txt`.

## What Scoutpost does

Scoutpost lets journalists create scheduled scouts that monitor public pages, news coverage, social profiles, civic sources, and watched transport activity. Each run extracts information units, deduplicates them, and lands them in an editorial inbox for verification.

## Key concepts

- **Scouts**: the scheduled monitoring jobs.
- **Information units**: atomic facts extracted from sources.
- **Verification**: the editorial checkpoint that promotes a lead into a citeable fact.
- **Credits**: the cost model for hosted runs.
- **Projects**: the container for scouts and exports.
- **Schedules**: Page Scouts can run daily; Beat and Civic Scouts run weekly or monthly; Fleet Scouts run every 3, 6, or 12 hours, or daily (satellites daily only).
- **Page Archive**: opt-in evidence snapshots for Page Scouts — retrieve via `scout snapshots` / `list_snapshots` / `GET /snapshots`; toggle via `archive_enabled`.

## Product behavior to preserve

- Page, Beat, Social, Civic, and Fleet scouts are the public concepts.
- Beat Scout may be scoped by topic or geography, but the geography-scoped flow is not a separate public surface.
- Beat Scout weekly is the highest supported frequency; daily Beat Scout schedules are rejected.
- Fleet Scout monitors a named watch list of vessels, aircraft, or satellites; it is not a broad traffic-search tool.
- Agents can help monitor, search, export, and summarize.
- Verification remains a hard boundary before publication.
- Hosted and self-hosted are both supported, but the hosted app is the primary public entrypoint.
